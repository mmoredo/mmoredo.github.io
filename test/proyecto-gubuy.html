Rediseñando la agenda de solicitud de turnos en la web de Uruguay

Proyecto Web de Uruguay (gub.uy)
Fecha Noviembre 2022
Equipo PM, Diseñadora UX

Resumen del proyecto

Nuestro objetivo fue mejorar la usabilidad y accesibilidad del formulario de solicitud de cédula de identidad en gub.uy, el sitio web oficial de Uruguay.

Para esto realizamos un análisis del formulario y luego priorizamos los problemas a solucionar por nivel de gravedad, proponiendo soluciones y mejoras.

Teniendo en cuenta las restricciones de tiempo, definimos hacer un rediseño del paso de solicitud del turno presencial, la agenda. En este paso encontramos las mayores barreras, y propusimos como mejora que el sistema muestre el primer turno disponible, además de dejar otras mejoras definidas para una próxima iteración.

Introducción

Gub.uy es el sitio donde las personas ciudadanas de Uruguay pueden encontrar información y servicios del estado.

Actualmente se está trabajando para mejorar la accesibilidad de todo el sitio, es por eso que nos pidieron analizar uno de los formularios de trámites de mayor relevancia, la solicitud de turno para gestionar el documento de identidad.

Nuestros objetivos fueron identificar las barreras de accesibilidad, mejorar la usabilidad y actualizar la estética a la librería de diseño que se utiliza actualmente en el sitio.

Desarrollo

Analizamos el formulario enfocándonos en la accesibilidad, compilamos una lista de errores y luego priorizamos los más críticos a remediar. Encontramos que en el paso de la agenda era donde estaban las mayores dificultades.

Como primera iteración, decidimos concentrar los esfuerzos en rediseñar por completo este paso y también actualizar la estética de todo el formulario.

Generamos una serie de propuestas en fidelidad baja, para ir explorando diferentes formas de presentar el contenido. Nos juntamos con la PM encargada de llevar el proyecto, para revisar, criticar el diseño y buscar mejoras aplicables a corto plazo.

Conservamos el orden de selección original: departamento, fecha y horario del formulario. También realizamos ajustes de diseño, como eliminar la preferencia de horario para mostrar los horarios disponibles a primera impresión.

Eliminamos el texto explicativo que hacía referencia a la selección del turno disponible mediante color verde, dado que no estamos incluyendo a las personas que no pueden percibirlo.

Encontramos que no hay posibilidad de conseguir turnos rápidamente, además de que hay pocas oficinas para realizar este trámite.

Es así que definimos como pequeña mejora que el sistema presente el primer turno disponible al seleccionar el departamento, para tener un acercamiento de solución a esta necesidad.

Dejamos definidas otras propuestas para una próxima iteración:

  • Permitir la selección del turno más próximo, o por fecha específica, sin necesidad de elegir un departamento en la agenda
  • Información extendida sobre la documentación a presentar
  • Simplificar la cantidad de pasos necesarios para realizar el trámite
  • Ayuda para completar el formulario
  • Incluir diseños de pantallas de cancelación y reprogramación de turnos

Nota: Este proyecto se entregó en fidelidad media por limitaciones de tiempo, con propósito de este portfolio se completó el prototipo en alta fidelidad.

Producto final

Conclusiones

Para validar el diseño, consideramos importante poder realizar pruebas de usabilidad con personas con ciudadanía y residentes de Uruguay. Esto nos ayudaría a ver las fortalezas y debilidades del diseño propuesto.

Nuestro rediseño presenta mejoras significativas en términos de accesibilidad y usabilidad que esperamos que contribuyan a brindar una mejor experiencia para todas las personas.
